常见的IE错误

1.操作终止（operation aborted）

当修改尚未加载完成的页面是，就会发生操作终止

上述代码，javascript代码位于div块里面，当执行到javascript代码的时候，在body的子元素的后面添加一个div子元素，但此时页面并未执行完成，即并未执行到div块完成，所以会导致错误，只要把script代码放在div块的外面就可以解决问题

2.无效字符（invalid character）

所谓的无效字符，就是指javascript语法未定义的字符，当出现这些字符时会抛出错误

3.未找到成员（重点）（Member not found）

IE中的所有DOM对象都是以COM对象，而非原声JavaScript对象的形式实现的。这会导致一些与垃圾收集相关的非常奇怪的行为。IE中未找到成员错误，就是由于垃圾收集例程配合错误所直接导致的。

具体来说，如果对象在被销毁之后，又给该对象赋值，就会导致未找到成员错误。而导致这个错误的，一定是COM对象，最常见的情形是使用event对象的时候

原因就是IE中的event对象是window属性，该对象在事件发生时创建，在最后一个事件处理程序执行完毕后销毁，上述代码在闭包中使用了event对象，但是却在1秒后执行，此时事件处理程序已执行完毕，导致event对象被销毁，所以就会发生未找到成员错误

4.未知运行错误

当使用innerHTML或outerHTML以下列方式指定HTML时，就会发生未知运行时错误（Unknown runtime error）：一是把块元素插入到行内元素时，二是访问表格任意部分（<table>，<tbody>等）的任意属性时。

5.语法错误

需要注意一种情况，当引用了外部的javascript文件时，该文件最终并没有返回javascript代码，就像<script>的src属性指向了HTML文件时。

6.系统无法找到指定资源

在使用javacript请求某个url，而该url的长度超过了IE对url最长不能超过2083个字符的限制时，就会发生错误
