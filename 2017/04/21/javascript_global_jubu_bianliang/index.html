javascript局部变量与全局变量

最近看到很多前端的面试题有js的全局变量还有局部变量的考题，以前以为懂了，但是看了题之后发现还没完全懂，所以就反复测试，百度，得到了一些总结。

首先看代码：

毫无疑问，上述会弹出先后弹出1和2。这是全局变量用在函数里面。

再来看代码

这次会弹出1，2，2。也就是说全局变量放在函数里面调用，并改掉其值，这会影响到全局变量的值，也就是在函数里面能够改变全局变量的值。

接下来到了最激动的时刻，看代码

你们觉得会输出什么，这里就有了惊天的变化，先后居然弹出1,2,2,undefined,4,2。本来一切都很顺利，但是这里居然弹了个undefined。原来，这是在test1()里面又定义了重名的局部变量a，把全局变量给冲刷掉，但是这个a只是定义了，并未赋值，所以先弹出undefined,然后给a赋值4，再弹出4，但是这个局部变量并没有影响到全局变量，这时全局变量还是2。而且这个局部变量a的定义是函数执行先优先进行的，执行test1()函数的时候，会首先浏览全体，首先知道a定义了，所以即使你的var a;和a=4;着两行互换，输出的结果还是一样的。
